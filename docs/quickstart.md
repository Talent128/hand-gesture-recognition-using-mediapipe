# 快速入门指南

## 5分钟快速开始

### 前提条件

- ✅ 已安装Anaconda/Miniconda
- ✅ 已安装Node.js（16+）
- ✅ 有可用的摄像头
- ✅ 已克隆项目到本地

### 一键启动

**Windows用户：**
```bash
cd hand-gesture-recognition-using-mediapipe
.\gesture_control_app\run.bat
```

**Linux/Mac用户：**
```bash
cd hand-gesture-recognition-using-mediapipe
chmod +x gesture_control_app/run.sh
./gesture_control_app/run.sh
```

脚本将自动完成：
1. 检测/创建conda环境（handGR）
2. 安装所有依赖
3. 启动后端服务（端口5000）
4. 启动前端服务（端口3000）
5. 打开浏览器

### 首次使用

#### 步骤1：选择功能

进入系统后，你会看到三个功能卡片：
- **PPT控制**：用手势控制PPT演示
- **视频控制**：用手势控制视频播放
- **系统设置**：配置手势映射

#### 步骤2：PPT控制演示

1. 点击"PPT控制"
2. 点击"上传PPT"或选择现有PPT（建议使用PDF格式）
3. 点击"启动摄像头"
4. 将手伸到摄像头前
5. 做出手势：
   - 👎 **大拇指向下** → 下一页
   - 👍 **大拇指向上** → 上一页
   - ✋ **张开手掌** → 第一页
   - ✊ **握拳** → 最后一页

#### 步骤3：视频控制演示

1. 点击"视频控制"
2. 点击"上传视频"或选择现有视频（建议MP4格式）
3. 点击"启动摄像头"
4. 做出手势：
   - ✋ **张开手掌** → 播放
   - ✊ **握拳** → 暂停
   - 👌 **OK手势** → 重新开始
   - ✌️ **V字手势** → 全屏
   - 👍/👎 **大拇指** → 音量调节

### 手势识别技巧

#### 最佳实践

1. **距离**：与摄像头保持40-80厘米
2. **光线**：确保环境光线充足
3. **背景**：选择简单背景，避免复杂图案
4. **手势**：做手势时保持1-2秒，让系统识别
5. **清晰**：手部完整出现在画面中

#### 常见手势示例

**静态手势：**

```
✋ Open（张开手）     ✊ Close（握拳）      👉 Pointer（指向）
   五指张开              手握成拳            食指伸出

👌 OK（OK手势）       ✌️ Peace（V手势）   👍 Thumbs Up
   拇指食指成圆          食指中指成V         大拇指向上

👎 Thumbs Down
   大拇指向下
```

**动态手势：**

```
⬆️ Move Up           ⬇️ Move Down        ⬅️ Move Left
   手向上移动            手向下移动           手向左移动

➡️ Move Right        🔄 Clockwise        🔃 Counter Clockwise
   手向右移动            顺时针画圆           逆时针画圆
```

### 自定义配置

#### 修改手势映射

1. 进入"系统设置"
2. 选择"PPT控制配置"或"视频控制配置"
3. 在下拉框中选择想要的操作
4. 点击"保存配置"

#### 修改键盘快捷键

1. 在设置页面找到"键盘快捷键映射"部分
2. 输入新的快捷键（如：`ArrowRight`、`Space`等）
3. 点击"保存配置"

### 故障排查

#### 问题1：摄像头无法启动

**可能原因：**
- 浏览器未授予摄像头权限
- 摄像头被其他程序占用
- 使用HTTP而非HTTPS（某些浏览器要求）

**解决方案：**
1. 检查浏览器地址栏，允许摄像头权限
2. 关闭其他使用摄像头的程序（如Zoom、Teams）
3. 使用Chrome或Edge浏览器

#### 问题2：手势识别不准确

**可能原因：**
- 光线不足
- 手部距离不当
- 背景复杂

**解决方案：**
1. 打开房间灯光
2. 调整与摄像头距离（40-80cm最佳）
3. 选择简单背景或移动到墙边
4. 做手势时保持清晰和稳定

#### 问题3：后端服务无法启动

**可能原因：**
- 端口被占用
- 依赖未安装
- Python版本不兼容

**解决方案：**
```bash
# 检查端口占用
netstat -ano | findstr :5000  # Windows
lsof -i :5000                 # Linux/Mac

# 重新安装依赖
conda activate handGR
pip install -r requirements.txt
pip install -r gesture_control_app/backend/requirements-backend.txt
```

#### 问题4：前端服务无法启动

**可能原因：**
- Node.js版本过低
- 依赖未安装
- 端口被占用

**解决方案：**
```bash
# 检查Node版本（需要16+）
node --version

# 重新安装依赖
cd gesture_control_app/frontend
rm -rf node_modules package-lock.json
npm install
```

### 停止服务

**使用启动脚本启动的：**
- 关闭弹出的命令行窗口即可

**手动启动的：**
- 在各自的终端按 `Ctrl + C`

### 提示与技巧

💡 **提示1：键盘快捷键**
系统同时支持手势和键盘控制，你可以两者结合使用。

💡 **提示2：文件格式**
- PPT建议转换为PDF格式，浏览器兼容性更好
- 视频建议使用MP4格式（H.264编码）

💡 **提示3：性能优化**
如果识别速度慢，可以：
- 关闭其他浏览器标签
- 降低视频质量
- 使用有线连接而非Wi-Fi

💡 **提示4：演示技巧**
进行PPT演示时：
- 提前测试手势
- 准备备用键盘
- 保持手势简洁明确
